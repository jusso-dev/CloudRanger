# ADR-0004: MCP-only surface (no web UI, no REST API, no scheduler)

**Status:** accepted · 2026-07-12

## Decision

The product surfaces are: the MCP server (agents) and a thin CLI
(operators). No web console, no REST API, no built-in scheduler or worker
fleet.

## Rationale

Operator decision: the product is "driven by Claude or Codex via MCP".
Reports are generated by the agent from `report_data` JSON; scheduling is
delegated to the agent platform (scheduled agent runs invoking the scan
prompts). Removing the web/REST/queue stack eliminates the largest attack
and maintenance surfaces.

## Consequences

- All business logic lives in shared packages (`engine`, `db`), so a future
  HTTP surface could be added without duplicating logic.
- Repeatability of exec reports comes from `report_data`'s defined metrics,
  not from server-rendered templates.
